\# Mitwirken an Leap (Contributing Guidelines)



Herzlich willkommen! Wir freuen uns, dass du Leap verbessern möchtest. Damit dein Pull Request (PR) akzeptiert wird, beachte bitte folgende Regeln:



\## 1. Erst ein Issue, dann der Code

Bevor du größere Änderungen vornimmst, öffne bitte ein \*\*Issue\*\* auf GitHub. Beschreibe dort, was du ändern möchtest. So vermeiden wir doppelte Arbeit und können das Design vorab besprechen.



\## 2. Code Style \& Qualität

Leap steht für Leichtigkeit und Klarheit.

\* Halte dich an die bestehende Syntax (siehe `docs.html`).

\* Kommentiere komplexe Logik (wie im Interpreter).

\* Achte auf saubere Variablennamen.



\## 3. Tests sind Pflicht

Da wir Playwright für die Website-Tests nutzen, stelle sicher, dass:

\* Bestehende Tests (`npx playwright test`) erfolgreich durchlaufen.

\* Neue Features durch eigene Test-Skripte im Ordner `tests` abgesichert sind.



\## 4. Lizenz-Hinweis

Mit dem Einreichen deines Pull Requests erklärst du dich damit einverstanden, dass deine Beiträge unter der \*\*CC BY-NC 4.0\*\* Lizenz veröffentlicht werden.



Vielen Dank für deine Unterstützung!
